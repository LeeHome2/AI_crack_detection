## 장면 3 — 진행상황 = 라이브 데모 (1:30 → 3:00) ★핵심

**[화면]** 폰으로 배포 URL 접속 화면을 그대로 녹화. 순서대로:

1. **원거리/흐린 사진** 업로드 → "📸 다시 촬영이 필요해요" 카드.
2. **근접 균열 사진** 업로드 → 위험도 등급 카드 + 탐지 이미지(빨간 박스+노란 중심선).
3. 스크롤: **📚 안전기준 근거(RAG)** — 근거 문장 + 출처 + 유사도.
4. 스크롤: **점검 보고서 초안 6섹션** + .md 다운로드 버튼.

**[대사]**
"실제로 폰에서 배포된 서비스를 보겠습니다. 먼저 멀리서 찍은 사진을 넣으면 — 보시는 것처럼 **AI가 '더 가까이 다시 촬영하라'고 게이트에서 막습니다.** 이번엔 균열을 근접 촬영한 사진을 넣겠습니다. **위험도 등급이 나오고**, 탐지된 균열이 박스와 중심선으로 표시됩니다. 아래로 내리면 — 판정만 하는 게 아니라 **'왜 그렇게 판단했는지' 안전기준 문서에서 검색한 근거를 출처와 함께** 보여줍니다. 그리고 마지막으로, **시설물 기본현황부터 종합의견까지 6개 섹션으로 된 점검 보고서 초안**이 자동 생성되고, 파일로 내려받을 수 있습니다. 원천 사진에서 최종 보고서까지 한 번에 이어지는 흐름입니다."
