# 요구사항 명세서 — AI 균열 안전점검 시스템

> 1인 개발 이호민 · 씨엠티정보통신·가천 부트캠프 · 최신화 7/22.
> 범위(공식 과제 9번): 비전문가가 시설물 사진을 올리면 **복합 결함(균열·철근노출·누수·파손)** 을 탐지·위험도 판정 + 근거(RAG) + 점검 보고서 초안을 생성하는 모바일 웹 자가진단 서비스.
> 표기: **1차 MVP** = 균열 전용(구현·배포 완료) / **2차 MVP** = 복합 결함(공식 스펙 충족) / **고도화** = 트리아지·멀티턴·세그.

## 3. 제약사항 (Constraints)

- **mm 절대측정 불가**: 참조객체 없는 단일 사진으론 실제 mm 폭 확정 불가 → 픽셀 상대값 사용. (고도화에서 mm 폭 라벨 데이터로 구간 추정 시도)
- **대상 고정**: 시설물 '건물 콘크리트'로 대상 종류 고정(도로·간판 등 제외). 결함은 복합(균열·철근노출·누수·파손) — 공식 스펙. 1차 MVP는 균열 전용, 2차 MVP에서 복합 확장.
- **탐지 지표 한계**: 가는 대각선 균열 + 사각형 박스의 IoU 한계로 mAP 절대값 낮음 → "균열 유무 + 시각 품질"로 병행 평가.
- **1인 개발·10일**: 산출물은 핵심 위주로 경량화, GPU 학습은 데스크탑(RTX 3070 Ti)에서만.

## 4. 사용자·이해관계자

- **주 사용자**: 비전문가(건물주·세입자·일반인) — 균열 자가진단.
- **간접 사용자**: 점검 실무 참고(보고서 초안을 전문가가 검토·확정).
- **운영자**: 개발자 1인(배포·모델·문서).
